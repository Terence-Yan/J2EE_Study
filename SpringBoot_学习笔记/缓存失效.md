#### 在SpringBoot项目中通过注解方式使用本地缓存，但缓存不生效
```
问题背景：
在如下的业务类中通过注解使用本地缓存
   @CacheConfig(value="myCache")
   public class MyService{
       ...
       @Cacheable
       public MyResult getData(param1, param2, param3){
           ......
       }
       ...
   }
问题现象：本地测试时，多次访问同一个请求，每次控制台都会输出数据库执行的相关日志，表明缓存并没有生效。
解决现场：通过debug调试，发现在LocalCache对象中存在缓存数据(有多个，很长时间没注意到这个问题)。后来，在分析代码的过程中，
         终于在getData的方法参数上发现了问题，getData方法原本有多个参数(超过5个)。后来，再一次代码整改过程中，将所有的
         业务参数封装成了一个业务对象(记为PojoA)，即getData方法的第一个参数param1，而param2, param3是分页相关的两个 
         int 型参数。而在项目中，缓存的 key 的生成方式采用的是默认的生成方式，即方法的所有参数有序组成的，所以，在检索
         缓存的时候，检查待查找的key是否存在，就要逐一比较被调用方法的各个参数是否相等，但由于在编写PojoA类时，没有重写
         equals方法，所以导致了缓存失效问题的发生：由于每次传递给getData方法的业务参数都是new的，
         所以，即使所有的业务参数都相等，但两个PojoA对象也是不同的。
解决方案：重写PojoA类的equals方法(需要同时重写hashCode方法)。
验证现场：本地测试时，多次访问同一个请求，只有第一次控制台会输出数据库执行日志，之后的请求，控制台均未输出数据库查询相关日志，
         表明缓存生效。   
```
